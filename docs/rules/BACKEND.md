# 백엔드 구현 규칙

## 기술 기준

- Java 21
- Spring Boot 3.x
- Maven
- Spring Web
- Spring Data JPA
- Flyway
- Bean Validation
- springdoc-openapi

## 구현 원칙

- Lombok을 쓰지 않는다.
- 명시적 생성자와 메서드를 사용한다.
- 컨트롤러는 HTTP 입출력만 담당한다.
- 트랜잭션 경계는 service 계층에 둔다.
- 엔티티 연관관계는 필요한 경우만 로딩한다.
- 기본은 지연 로딩이다.

## 패키지 규칙

- 도메인 기준 패키지 구성을 유지한다.
- 새 코드도 가능한 한 기존 도메인 패키지에 넣는다.
- 공통 규칙과 공통 예외는 `common`에 둔다.
- 환경 설정은 `config`에 둔다.

## 변경 원칙

- 과도한 일반화보다 현재 게임 규칙에 맞는 구현을 우선한다.
- 새 규칙이 생기면 먼저 `docs/`를 갱신한다.
- 작업 중 구조를 바꿨다면 이유가 문서에 남아야 한다.
