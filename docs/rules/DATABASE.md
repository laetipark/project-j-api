# 데이터베이스 규칙

## 기본 원칙

- MySQL 8 기준으로 설계한다.
- 스키마 변경은 Flyway migration으로만 관리한다.
- 로컬 개발과 테스트 모두 Flyway를 기준으로 맞춘다.

## 모델링 원칙

- 카탈로그 테이블은 `code` unique를 기본으로 둔다.
- 동적 규칙 값은 시드 데이터로 관리한다.
- 업그레이드 가능 여부는 계산 결과이지 영구 상태 컬럼이 아니다.
- `player_region_unlocks` 같은 영구 해금 테이블은 만들지 않는다.

## 저장 규칙

- `selected_recipe_id`는 nullable 유지
- `player_tools`는 영구 해금 저장
- inventory slot 계산은 “종류 수”로 수행
- 로그 테이블은 조회보다 이력 보존을 우선한다

## SQL 작성 원칙

- 의미가 있는 제약 이름을 직접 붙인다.
- 초기값과 밸런스 값은 migration seed에서 관리한다.
- 게임 기획이 바뀔 수 있는 값은 코드 상수에 숨기지 않는다.
