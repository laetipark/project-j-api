# 커밋 메시지 작성 규칙

이 문서는 이 저장소의 Git 커밋 메시지 규칙 정본이다.
다른 문서에서는 이 문서만 참조한다.

## 1. 기본 원칙

- 모든 Git 커밋 메시지는 한국어 한 줄로 작성한다.
- 형식은 `타입 : 내용`이다.
- 전체 한 줄은 50자 이내로 유지한다.
- `타입`은 아래 표의 값만 사용한다.

## 2. 절대 출력 규칙

- 커밋 메시지를 만들 때는 이 저장소 규칙만 따른다.
- 출력은 한국어 한 줄만 허용한다.
- 본문, 불릿 목록, 설명 문장, changelog 요약은 금지한다.
- 형식은 반드시 `타입 : 내용`만 허용한다.
- 50자를 넘기면 실패로 간주하고 다시 줄인다.
- 영어 문장형 제목, `docs: ...` 같은 콜론 형식, 자동 생성 diff 요약은 금지한다.
- 제목 끝에 마침표를 붙이지 않는다.
- 파일 경로, 코드 식별자, 브랜치명처럼 번역하면 안 되는 고유 명칭 외에는 영문을 쓰지 않는다.

## 3. 타입 표

| 타입 | 의미 |
| --- | --- |
| `feat` | 새로운 기능 추가 |
| `update` | 기존 기능 조정 |
| `fix` | 버그 수정 |
| `docs` | 문서 또는 주석 수정 |
| `design` | UI 또는 시각 표현 변경 |
| `style` | 동작 변화 없는 형식 수정 |
| `rename` | 파일 또는 식별자 이름 변경 |
| `delete` | 불필요한 파일 또는 코드 삭제 |
| `refactor` | 동작 유지 전제의 구조 개선 |
| `test` | 테스트 추가 또는 개선 |
| `chore` | 빌드, 설정, import 정리 등 유지보수 |

## 4. 형식

```text
타입 : 내용
```

## 5. 예시

- feat : 허브 팝업 흐름 정리
- fix : 폐광 포털 잠금 안내 수정
- docs : .aiassistant 문서 체계 정리

## 6. 금지 예시

- `docs: Add agent workflow and source-of-truth rules to project guide`
- 여러 줄 본문이 붙은 커밋 메시지
- 영문 bullet 요약이 붙은 커밋 메시지
- `타입 : 내용` 앞뒤에 설명 문장이 추가된 출력

## 7. 커밋 메시지 생성 입력 템플릿

커밋 메시지를 생성할 때는 아래 지시를 그대로 우선 적용한다.

```text
이 저장소 규칙만 따를 것.
한국어 한 줄만 출력할 것.
본문, 불릿, 설명, changelog 출력 금지.
형식은 타입 : 내용만 사용할 것.
전체 50자 이내로 작성할 것.
```

## 8. 템플릿 운영 규칙

- 이 파일은 로컬 `commit.template`의 정본이다.
- 커밋 규칙을 바꾸면 엔트리 파일과 관련 project 문서의 링크도 함께 확인한다.
