# 테스트 규칙

## 기본 전략

- 규칙이 있는 기능은 통합 테스트를 우선한다.
- H2 MySQL 호환 모드와 Flyway로 테스트 환경을 맞춘다.
- 상태 전이와 자원 변동은 snapshot이나 저장소 조회로 검증한다.

## 최소 검증 대상

- phase 전이
- service 계산
- inventory slot 규칙
- storage 이동
- upgrade 구매
- portal 접근 조건

## 테스트 작성 원칙

- 한 테스트는 한 규칙을 명확히 드러내야 한다.
- 밸런스 값 자체보다 계산 규칙을 검증한다.
- 회귀가 잦은 흐름은 end-to-end에 가까운 서비스 호출로 검증한다.
